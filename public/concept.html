RPC API Concept Notes

Build method-oriented APIs with strict request envelopes and structured errors. ← Back · Practice

RPC model

  • Client calls named method
  • Server validates params and executes procedure
  • Response contains result or error object

Design concerns

  • Method naming and versioning strategy
  • Auth at method boundary
  • Consistent error codes and structure
  • Idempotency behavior per method
